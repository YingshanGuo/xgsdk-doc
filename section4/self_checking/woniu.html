测试包检查
分类 系统版本 测试点 测试点详细
版本信息检查 android 版本号version code，version name version code 必须为整数
version code、version name需要不为0,需只含数字和“.”（小数点）
主包名package 新游戏提交测试，package字段需要区别于其他渠道，带上“snail”字样,package="com.app.snail"
更新版本时，package字段需要与已上架版本一致（区分大小写）
Android版文件名命名格式：
gamename_v1.0_201111171103.apk
例如：超级玛丽1.5版是2011年8月1号早上8点10分打包提交的，那么文件名就是：SuperMario_v1.5_201108010810.apk
同一游戏的不同测试包，package字段需要有所区别
图标检查 android 图标规范 游戏的icon图标必须加入免商店logo，且角标大小需与上架资料中提供的一致
游戏中配置的所有icon文件以及提供的icon文件均需打上免商店logo，并且保持一致
支付检查 android 从游戏的商品列表进入到免商店支付界面 传入价格需保持一致
确保充值页面能正常打开，并连接到蜗牛手机充值方式的选择界面（包括：支付宝、财付通等）
在游戏内商城充值消费，道具需在1分钟内到账。若无法及时到账或到账的代币数量不正确，需要添加弹框有相关文字说明。。通知告知玩家如何操作或添加刷新按钮。（请用带中文的账号测试）
商品名称、数量需要根据购买项有区别，并且注明代币名称。
如：名称是10个金币，数量是1；或者名称是金币，数量是10。
游戏商品进行促销、额外赠送等方式，需有相应的列表提示。
如：
游戏商品列表写上10金币，赠送1金币，免商店支付界面写10金币（正确）
游戏商品列表直接写11金币，免商店支付界面写10金币（错误）
在免商店支付界面支付完成回到游戏中 道具需在3-5分钟内到账
道具到账后必须能及时刷新，无需重启游戏；若无法自动刷新，需弹框提示，告知玩家如何操作或者添加手动刷新按钮
到账的道具数量正确。若有充值活动导致到账的代币数量不正确，需在公告或者是商品列表中有相关文字说明
使用代币充值的游戏，需注意：
1）在牛币兑换界面修改支付的牛币数量。支付成功后，到账的代币数量需与支付的牛币比例一致；
2）需注意牛币与代币的兑换比例，必须可以整除.
玩家牛币不足或者进入支付界面不进行购买的情况，需保证回到游戏界面，可以继续游戏，不会持续loading，不会崩溃。
订单号要求 订单号需只含有数字、字母、下划线、减号，且不能重复
不含免商店支付方式 游戏中需不含有任何非免商店的充值方式以及非免商店充值方式的介绍（如活动、公告界面；帮助界面），即使是点击无响应的按钮（必须屏蔽相关按钮）；
游戏界面内显示的内容、调用通讯工具或社交软件功能的默认发送内容需不含免商店引导 需不含引导用户到其他下载地址下载非免商店渠道的客户端（包括官网）
需不含引导用户使用非免商店支付方式的相关信息
需不含引导用户到其他游戏平台进行好评，给予奖励
需不含引导用户访问非免商店官网、论坛的相关信息
需不含官网、官方论坛等类似访问官方网站按钮，即使是灰状的，无法点击的
需不含免商店渠道用户无法参与的分享活动
需不含微博分享按钮，点击后进入到官方微博主页
需不含任意第三方开发平台以及第三方广告平台系统（商务授权的除外）
不含免商店客服QQ 游戏中必须保证不含免商店的对外客服QQ
客服信息一致性 游戏中出现的客服邮箱、客服电话需同之前上架资料表中提供的以及dev后台配置的相同
若更换客服信息，需要重新提供上架资料表，并且修改dev后台配置的客服信息。
非中文检查 android 游戏名称检查 安装后手机桌面的名称、dev后台的名称需为简体中文
按钮检查 需保证不含大面积的按钮均为英文
帮助信息检查 帮助界面的文字，或者是帮助图片上的文字需是简体中文的
其他检查 android 游戏安装包的固件要求必须写 游戏安装包的固件要求需配置
即需要设置AndroidManifest.xml文件中的minSdkVersion或者targetSdkVersion
androids 敏感词屏蔽 游戏中任意可以输入文本的地方，均需对钓鱼岛、尖阁列岛、习近平、周永康、薄熙来、习仲勋敏感词的发送进行屏蔽，详情见屏蔽字
游戏名称一致 新游戏的游戏名称需与游戏安装后的游戏名称一致
